# Regras & Assinaturas do Log

## Introdução

Para ajudar meu estimados colegas, crio então este arquivo markdown para que possamos documentar de forma **organizada** alterações no projeto e suas **explicações**. Ao assinarem seus nomes, os participantes aceitam que comentários realizados nos arquivos de código devem ser curtos, sem explicações longas e desnecessárias. O máximo que será aceitado serão comentários '''assim''' após a definição de uma função para explicar em poucos paragrafos o argumento recebido, o que faz e o que retorna.

Aquele que desrespeitar as regras ditadas acima estará condenado a pagar uma bebida energética para cada um dos demais integrantes por um determinado período de tempo. Esse período será decidido pelos membros com base na gravidade do caso.
